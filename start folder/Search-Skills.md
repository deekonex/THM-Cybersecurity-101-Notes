# Search Skills

## Core Research Philosophy: Trust, but Verify

The quality of intelligence depends on the research method. Critical evaluation is mandatory.

**Source Authority:** Always check who wrote it. Official documentation carries more weight than random blogs.

**Objectivity/Bias:** Identify agenda. Be wary of content pushing a specific product or commercial angle.

**Evidence & Logic:** Facts must be backed by credible evidence, not merely presented as opinion.

**Corroboration:** The strongest evidence comes from multiple, independent, and reliable sources agreeing on the same facts.

---

## Advanced Search Engine Operators (Google)

Basic searching is insufficient. These operators refine and narrow results dramatically.

**Exact Phrase:** Use double quotes (" "). Guarantees the search engine finds the phrase in the exact order specified.
- Example: `"passive reconnaissance"`

**Site Restriction:** Limits search results to a single domain.
- Format: `site:domain.com term`

**Exclusion:** Uses the minus sign (-) to eliminate irrelevant results.
- Format: `term -irrelevant_term`

**File Type Search:** Restricts results to a specific file format.
- Format: `filetype:pdf term`

## Specialized Intelligence Engines

These platforms go beyond general search and provide specialized, actionable data.

**Shodan:** A search engine for internet-connected devices (IoT, servers, hardware). Essential for finding devices running specific services.

**Censys:** Focuses on enumerating hosts, domains, and certificates. Used for auditing open ports and discovering network assets.

**VirusTotal:** Malware analysis platform. Upload files, URLs, or file hashes to check for detection by dozens of antivirus engines.

**Have I Been Pwned (HIBP):** Checks if a given email address has been compromised in a known data breach.

## Vulnerability & Technical Documentation

These standards and tools provide verifiable technical knowledge.

**CVE (Common Vulnerabilities and Exposures):** The standardized dictionary for publicly known vulnerabilities. Assigns a unique ID (e.g., CVE-2024-3094).

**Exploit Database / GitHub:** Primary resources for finding tested exploit code (Proof-of-Concept, or PoC) related to specific CVEs.

**Technical Manuals:** Always refer to official product documentation (e.g., Microsoft, Apache).
- **Linux:** Use the man command (e.g., `man cat`)
- **Windows:** Use official Microsoft technical documentation for command syntax.

---

## Social Media & Information Gathering

Treat social platforms as both valuable research tools and high-risk data sources.

**LinkedIn:** Valuable for professional due diligence. Ideal for researching an employee's technical career history.

**Public Social Media (Facebook/etc.):** High risk of oversharing. Often the source of sensitive security answers.

**News Outlets:** Monitor reputable news sources and security experts for timely threat intelligence.
